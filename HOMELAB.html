Homelab Project Documentation

Objective

The primary objective of the homelab project was to expand my knowledge of IT systems and fundamental IT concepts. Additionally, I aimed to establish an environment conducive to further learning in areas such as programmable logic controllers (PLCs), IoT devices, and the integration of electrical and computer systems. A key constraint of the project was to utilize only free software and hardware resources available to me.


Hardware and Software Configuration

For the project, I utilized my personal computer, equipped with an AMD Ryzen 3600X processor, 16 GB of DDR4 RAM operating at 2133 MHz, and a 1 TB SSD with approximately 300 GB allocated for the project's storage needs. Given the limitation of available hardware, virtualization was chosen as the primary means to create the lab environment. Hyper-V was employed as the virtualization platform, enabling the installation of Ubuntu Core and Windows Server 2022 operating systems.


Deployment Process

After enabling virtualization in the BIOS and configuring Hyper-V on the host system, the deployment process involved installing the requisite software and carefully following installation procedures outlined in the documentation. Each virtual machine (VM) was provisioned according to the recommended system requirements, with additional resources allocated to ensure optimal performance.


Networking Setup Challenges

Despite successful VM deployment, challenges arose during the networking setup phase, particularly with the implementation of pfSense as a firewall/router solution. An issue emerged with the WAN switch within Hyper-V not obtaining an IP address from the home router/modem, rendering upstream connectivity impossible. Extensive troubleshooting efforts were undertaken, including consultation of pfSense documentation and online forums. However, these attempts proved unsuccessful, ultimately leading to a halt in the project due to network interface card failures on the host system.


Lessons Learned

The project highlighted the importance of resilience and adaptability in the face of technical challenges. It became evident that a balanced approach to problem-solving is essential, as excessive focus on resolving specific issues can hinder progress in other project areas. Moving forward, I intend to refine my time management skills and adopt a more strategic approach to troubleshooting.


Future Considerations

In future iterations of the project, I aim to address hardware limitations by acquiring a secondary system with dedicated networking capabilities. Alternatively, exploring alternative networking solutions better suited to my current skill level may present a viable path forward. Despite encountering setbacks, the project has provided valuable insights into VM deployment and management, laying the groundwork for future endeavors in IT infrastructure development.


Conclusion

In conclusion, while the homelab project faced challenges in network configuration, it served as a valuable learning experience in IT systems administration and virtualization. Through diligent documentation and reflection, I have gained valuable insights into the complexities of networking environments and the importance of resource management in project planning.


References

Documentation from the pfSense website, along with various online resources including Stack Overflow and Reddit, served as valuable references throughout the project.
